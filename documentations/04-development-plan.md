# План разработки по Agile

| Этап | Работа | Ответственный | Результат |
|---|---|---|---|
| Идея | Определить назначение DayLine и основные модули | Оба | Выбраны список дел, питание и финансы |
| Планирование | Записать требования, User Story и распределить задачи | Оба | Составлен план первой версии |
| Разработка | Создать основу, главную страницу, сервер и подключение к базе | Оба | Сайт запускается локально |
| Разработка модулей | Сделать календарь и список дел | Эрдэм | Задачи сохраняются по датам |
| Разработка модулей | Сделать регистрацию, питание, КБЖУ и финансы | Валентин | Работают аккаунты, продукты и операции |
| Тестирование | Проверить модули друг друга | Оба | Заполнены чек-листы и тест-кейсы |
| Исправление дефектов | Исправить ошибки, записанные тестировщиком | Автор модуля | Исправленная версия возвращена в тестирование |
| Релиз | Объединить готовые модули и разместить сайт | Оба | Версия DayLine работает на Render |
| Сбор обратной связи | Проверить сайт на телефоне и собрать замечания | Оба | Составлены задачи для следующего этапа |

После исправления дефекта модуль возвращался на тестирование. После релиза обратная связь использовалась при следующем планировании.
